# Chinese Writing Kit Instructions

## 一般原则

1. 本规范仅适用于 Markdown 和 MDX 文件中的**正文内容**，不对以下元素进行检查或修改：
   - Frontmatter（YAML 头部元数据）
   - 代码块（code blocks）及行内代码（inline code）的内部内容
   - MDX 组件标签及其属性（如 `<Component prop="value" />`）
   - HTML 标签及其属性
   - URL、文件路径、变量名等技术性字符串
   - 数学公式（LaTeX）

2. Markdown 语法元素的处理：
   - 链接文字 `[文字](/)`、图片替代文字 `![alt](/)` 中的文字部分应遵循本规范。
   - 标题、列表项、表格单元格中的文字内容应遵循本规范。
   - 强调标记（`**加粗**`、`*斜体*`）内的文字应遵循本规范。
   - 引用块（`>`）中的内容应遵循本规范，包括标点符号、中英文混排等规则。

3. 保持原有格式的情况：
   - 引用他人原文时，如需保留原文风格，可在引用块前注明「以下为原文」或类似说明。
   - 专有名词、品牌名称保持其官方写法。
   - 代码相关的术语（如函数名、命令）保持原样。

## 错别字

- 检查并纠正常见的可能的中文错别字，忽略出现在「豁免」列表中的词语。
- 豁免以下的词语：（暂无）

## 标点符号

1. 在中文文段中使用以下的全角标点符号：

- 句号（。）
- 逗号（，）
- 顿号（、）
- 分号（；）
- 冒号（：）
- 引号（「」和『』）
- 括号（（））
- 问号（？）
- 感叹号（！）
- 省略号（……）
- 破折号（——）
- 书名号（《》）

2. 在英文文段中使用以下的标点符号（引号使用弯引号，即 smart quotes）：

- 句号（.）
- 逗号（,）
- 分号（;）
- 冒号（:）
- 引号（“” 和 ‘’）
- 括号（()）
- 问号（?）
- 感叹号（!）
- 省略号（...）
- 破折号（--）

3. 问号用法：

- 选择问句中，通常只在最后一个选项的末尾用问号，各个选项之间用逗号隔开。
  - ✓ 这是巧合还是有意安排？
  - ✗ 这是巧合？还是有意安排？
- 当含有疑问词的语段充当句子成分，而句子并不表示疑问语气时，句末不用问号。
  - ✓ 我不知道他去了哪里。
  - ✗ 我不知道他去了哪里？

4. 顿号用法：

- 用于并列词语之间的停顿。
  - ✓ 这里有自由、民主、平等、开放的风气。
- 标有引号的并列成分之间、标有书名号的并列成分之间通常不用顿号。
  - ✓「日」「月」构成「明」字。
  - ✓《红楼梦》《三国演义》《西游记》是中国古典名著。
  - ✗「日」、「月」构成「明」字。
- 相邻或相近两数字连用表示概数时通常不用顿号。
  - ✓ 三五成群、八九公里、十七八岁
  - ✗ 三、五成群
- 用阿拉伯数字表示年月日的简写形式时，用短横线连接号，不用顿号。
  - ✓ 2024-03-02
  - ✗ 2024、03、02

5. 分号用法：

- 表示复句内部并列关系的分句之间的停顿（尤其当分句内部还有逗号时）。
  - ✓ 语言文字的学习，就理解方面说，是得到一种知识；就运用方面说，是养成一种习惯。
- 用于分项列举的各项之间。
  - ✓ 岗位职责为：一、讲授主干课程；二、主持科研项目；三、领导学术队伍建设。
- 分项列举的各项有一项或多项已包含句号时，各项的末尾不能再用分号。
  - ✓ 一是建立生产服务体系。成立服务公司，提供机耕服务。二是建立销售体系。……
  - ✗ 一是建立生产服务体系。成立服务公司，提供机耕服务；二是建立销售体系。……

6. 冒号用法：

- 用于总说性或提示性词语之后，表示提示下文。
  - ✓ 北京紫禁城有四座城门：午门、神武门、东华门和西华门。
- 冒号提示范围无论大小，都应与提示性话语保持一致（即在该范围的末尾要用句号点断）。
  - ✓ 艾滋病有三个传播途径：血液传播、性传播和母婴传播。日常接触不会传播。
  - ✗ 艾滋病有三个传播途径：血液传播、性传播和母婴传播，日常接触不会传播。
- 一个句子内部一般不应套用冒号。
- 冒号应用在有停顿处，无停顿处不应用冒号。
  - ✓ 他头也不抬，冷冷地问：「你叫什么名字？」
  - ✓ 这事你得拿主意，光说「不知道」怎么行？（无停顿，不用冒号）

7. 引号用法：

- 当引号中还需要使用引号时，外面一层用双引号「」，里面一层用单引号『』。
  - ✓ 他问：「老师，『七月流火』是什么意思？」
- 独立成段的引文如果只有一段，段首和段尾都用引号；不止一段时，每段开头仅用前引号，只在最后一段末尾用后引号。

8. 破折号用法：

- 破折号的形式是「——」（占两个字位置）。
- 可用于标示注释内容、补充说明、插入语、总结上文或提示下文、话题的转换、声音的延长等。
  - 注释补充：一个矮小而结实的日本中年人——内山老板走了过来。
  - 话题转换：「好香的干菜，——听到风声了吗？」
  - 声音延长：「嘎——」传过来一声水禽被惊动的鸣叫。

9. 省略号用法：

- 省略号的形式是「……」（六连点）。
- 省略号和「等」「等等」「什么的」等词语不能同时使用。
  - ✓ 含有铁质的食物有猪肝、大豆、油菜、菠菜等。
  - ✓ 含有铁质的食物有猪肝、大豆、油菜、菠菜……
  - ✗ 含有铁质的食物有猪肝、大豆、油菜、菠菜……等。

10. 书名号用法：

- 用于标示书名、卷名、篇名、刊物名、报纸名、文件名，以及电影、电视、音乐、诗歌等作品的名称。
  - ✓《红楼梦》《人民日报》《沁园春·雪》
- 当书名号中还需要书名号时，里面一层用单书名号〈〉，外面一层用双书名号《》。
  - ✓《教育部关于提请审议〈高等教育自学考试试行办法〉的报告》
- 不能视为作品的课程、课题、奖品奖状、商标、证照、组织机构、会议、活动等名称，不应用书名号。
  - ✗ 下学期将开设《现代企业财务管理》课程。
  - ✗ 本市将召开《全国食用天然色素应用研讨会》。
  - ✓ 下学期将开设「现代企业财务管理」课程。

11. 连接号用法：

- 短横线「-」：用于化合物名称、门牌号码、电话号码、年月日简写、复合名词、产品型号等。
  - ✓ 010-88842603、2024-03-02、WZ-10 直升机、吐鲁番-哈密盆地
- 一字线「—」或浪纹线「~」：用于标示相关项目的起止（如时间、地域）和数值范围。
  - ✓ 沈括（1031—1095）、北京—上海、25~30 g、第五~八课

12. 标点符号位置：

- 句号、逗号、顿号、分号、冒号不出现在一行之首。
- 问号、叹号不出现在一行之首。
- 引号、括号、书名号的前一半不出现在一行之末，后一半不出现在一行之首。
- 破折号占两个字位置，不能中间断开分处上行之末和下行之首。
- 省略号占两个字位置，不能中间断开分处上行之末和下行之首。

13. 竖线用法：

- 总是使用半角竖线「|」，不要使用全角竖线「｜」。
  - ✓ 选项 A | 选项 B
  - ✗ 选项 A ｜ 选项 B

## 中英文混排

1. 中英文之间需要加空格（盘古之白）。
   - ✓ 在 GitHub 上进行代码托管。
   - ✓ 使用 Python 编写脚本。
   - ✓ 这是一个 test 测试。
   - ✗ 在GitHub上进行代码托管。
   - ✗ 使用Python编写脚本。

2. 中文与数字之间需要加空格。
   - ✓ 今天是 2025 年 12 月 21 日。
   - ✓ 这个项目有 128 个文件。
   - ✗ 今天是2025年12月21日。

3. 数字与单位之间需要加空格，但度数、百分比除外。
   - ✓ 文件大小为 10 MB。
   - ✓ 距离约 500 km。
   - ✓ 今天气温 25°C。
   - ✓ 完成度达到 80%。
   - ✗ 文件大小为 10MB。

4. 中文标点与英文或数字之间不加空格。
   - ✓ 我使用 GitHub，它很方便。
   - ✓ 共有 100 人参加，包括 50 名学生。
   - ✗ 我使用 GitHub ，它很方便。

5. 完整的英文句子或专有名词内部保持英文原有的空格规则，并强调使用弯引号（smart quotes）。
   - ✓ 文章标题为「How to Learn Programming」。
   - ✓ 安装 Visual Studio Code 编辑器。

6. 链接与前后中文之间始终加空格，链接内部内容仍遵循中英文混排规则。
   - ✓ 请访问 [GitHub](https://github.com) 获取更多信息。
   - ✓ 更多信息请参考 [文档](https://example.com)。
   - ✓ 这是 [VS Code 编辑器](https://code.visualstudio.com) 的官网。
   - ✓ 详见「[使用指南](https://example.com)」。（链接紧邻标点时，标点侧不加空格）
   - ✓ 访问 [文档](https://example.com) 了解更多信息。

7. 英文标点（包括弯引号）与中文之间不加空格，使用英文标点后按英文规则处理。
   - ✓ 他说「Hello, World!」是编程入门的第一课。

8. 行内代码与中文之间需要加空格。
   - ✓ 使用 `npm install` 命令安装依赖。
   - ✓ 将变量命名为 `userName`。
   - ✗ 使用`npm install`命令安装依赖。
